The "Quota" Effect: Good vs. Bad Annotation

When military units were forced to meet quotas (e.g., "label 3 images before login"), quality suffered. Inconsistent bounding boxes destroy model precision.

Good Annotation

High Consistency
  • Each object has its own box.
  • Boxes are "tight" to the object boundaries.
  • No overlap between the person and the car.

Bad Annotation

Quota Driven
  • Two people grouped in one box (confuses count).
  • Car box is too wide and overlaps the person.
  • Result: Model learns that "Cars" include "People".

Annotation Vitality

A method to measure annotator quality. We drop one annotator at a time and see how the model consensus changes.

The Golden Rule:

  • If consensus improves when dropped: Bad Annotator.
  • If consensus worsens when dropped: Good Annotator.

Key Insight: The Trade-off

Dropping Annotator 1 (the inconsistent one) gave the Highest Precision (0.70) and lowest errors (375), but the Lowest Recall (0.57) because the total data volume decreased.

"Precision requires consistency. Recall requires volume."

Precision

CORRECTNESS
TP / (TP + FP)

"When I say it's a vehicle, is it actually a vehicle?"

Driven by Consistency
Cannot be fixed if trained on bad data.

Recall

COMPLETENESS
TP / (TP + FN)

"Did I find ALL the vehicles in the image?"

Driven by Data Volume
Can be improved later by adding more data.

Strategies for Success

High Overlap First

Start pilots with 30-50% overlap (multiple annotators per image). Measure agreement strictly.

Project Maven Lesson

Active Learning

Train a model, find where it is uncertain, and only annotate those specific high-value images.

Augmentation

Use geometric flips, color jitters, or Generative AI to boost volume without losing consistency.

Key Principles Summary

1. Asymmetry of Recovery

You will naturally get more data (volume) over time. You cannot easily fix inconsistent data (quality) later.

2. Vitality Logic

Wrong annotations are worse than missing annotations. If removing a person improves the model, remove them.

3. Technical Debt

Inconsistent data is permanent debt. Fix it early or pay 10x later.
