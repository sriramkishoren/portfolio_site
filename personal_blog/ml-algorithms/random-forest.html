Random Forest Algorithm

An ensemble learning method that builds a "forest" of decision trees to improve accuracy, stability, and reduce overfitting.

How It Works: Step-by-Step

1

Bootstrap Sampling

Create multiple datasets by sampling with replacement from original data.

Key: ~63.2% unique data per sample

2

Build Trees

Grow a decision tree for each sample. Do not prune.

Key: Random subset of features at each split

3

Predict

Pass new data through every single tree in the forest. Each tree gives its own independent prediction.

4

Aggregate

Combine all predictions.

Classification: Majority Vote
Regression: Average

Technical Mechanics

Bagging (Bootstrap Aggregating)

The method of training models on different subsets of data to reduce variance.

  • Sampling with replacement
  • Reduces overfitting ("averaging reduces risk")

Feature Randomization

At every node split, the algorithm only considers a random subset of features.

Classification: √n_features
Regression: n_features / 3

Out-of-Bag (OOB) Error

Validation built into the training process. ~36.8% of data is not seen by a specific tree.

  • Acts as validation without separate test set
  • Unbiased error estimate

Feature Importance

Calculates which features contribute most to the predictive power.

Gini Importance (MDI) Fast, but biased toward high-cardinality features.
Permutation Importance Slower, more reliable. Shuffles data to test impact.

Key Hyperparameters

Parameter Description Impact
n_estimators Number of trees in the forest. More is better (up to a point), but slower.
max_features Max features considered per split. Controls diversity. Lower = more diverse trees.
max_depth Maximum levels in each tree. Controls complexity. Prevents overfitting.
min_samples_split Min samples required to split. Higher values = more generalized trees.

Pros & Cons

Advantages

  • + Reduces overfitting (vs single tree)
  • + Robust to noise and outliers
  • + Handles missing data well
  • + Good performance out-of-the-box

Disadvantages

  • - Requires tuning of additional parameters (e.g., # of trees)
  • - Computationally intensive (slow training/pred)
  • - Hard to interpret (Black Box)
  • - Large memory footprint
  • - Limitations: High Cardinality & Extrapolation (See Below)

Deep Dive: Critical Limitations

1. High Cardinality Bias

Random Forest struggles with categorical variables that have many unique values (High Cardinality), like UserIDs or Zip Codes.

The Splitting Problem

High cardinality features create extremely fragmented splits, leading to overfitting on rare categories.

The Importance Bias

Gini importance is biased toward features with more unique values because they offer more "split points."

2. No Extrapolation

Unlike linear regression, Random Forest cannot predict values outside the range of the training data. It will "plateau" at the boundaries.

The Problem

If you train on house prices $100k-$500k, the model can never predict $600k. It assigns the average of the closest "leaf" node, which is max $500k.

Algorithm Comparison

Random Forest

Best for general purpose tabular data.

  • ✓ Parallel Training
  • ✓ Robust to Overfitting
  • ✗ Slow Prediction

XGBoost/GBM

Sequential improvement.

  • ✓ Higher Accuracy Potential
  • ✗ Sequential (Slower Training)
  • ✗ Harder to Tune

Neural Networks

Unstructured data (Images, Text).

  • ✓ Handles Perceptual Data
  • ✗ Needs Massive Data
  • ✗ Black Box / Computation

When to use Random Forest?

  • Structured / Tabular Data
  • Medium to Large datasets (thousands of rows)
  • When you need Feature Importance insights
  • As a strong baseline model

Avoid When:

  • Real-time latency is critical (prediction is slow)
  • Data is very sparse / high-dimensional (text)
  • You need to predict values outside training range
