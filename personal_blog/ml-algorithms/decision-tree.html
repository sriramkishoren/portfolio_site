What is a Decision Tree?

A hierarchical model that allows you to make classification decisions through a series of questions. The algorithm selects features (variables like height, weight) and parameters (threshold values like 6'1", 200 lbs) to create decision rules that split data into increasingly pure groups.

Tree Structure Components

Root Node

Starting point of the tree. Contains the first and most important decision that best splits the entire dataset.

Decision Node

Internal nodes that evaluate a condition. Each decision splits data into subsets based on feature values.

Leaf Node

Terminal nodes containing final predictions. No further splitting occurs—classification or regression value is assigned.

Branch/Edge

Connections between nodes representing outcomes (Yes/No, True/False). Shows the path taken based on conditions.

Splitting Criteria & Metrics

Entropy

A metric used to evaluate the performance of a decision tree by measuring the randomness or disorder in the data. Originally developed in thermodynamics to measure disorder in physical systems.

H(S) = -Σ p(i) × log₂(p(i))

0 = Perfectly pure (all same class)
1 = Maximum impurity (50/50 binary)
Lower entropy = Better classification

Key Point: Entropy is NOT a measure of model complexity, NOT a visualization tool, and does NOT address class balancing. It specifically measures randomness/disorder to evaluate tree performance.

Gini Index

Measures the probability of incorrect classification when a random sample is classified according to the distribution.

Gini = 1 - Σ p(i)²

0 = Perfectly pure
0.5 = Maximum impurity (binary)
Faster to compute than Entropy

Information Gain

The reduction in entropy achieved after splitting. Higher information gain means the split provides more "information" about the target variable.

IG(S, A) = Entropy(parent) - Σ (|Sv|/|S|) × Entropy(Sv)

Advantages

Easy to Interpret

Tree structure is intuitive—can visualize on small examples. Accessible to non-technical stakeholders.

Handles Mixed Data Types

Works with both numeric and categorical data within the same model.

No Distribution Assumptions

Doesn't require assumptions about statistical distribution of the data.

Works with Small Data

Can build manually from hand and see how it performs, then tweak features or values.

Combines with Other Models

Can use decision tree for certain classes and more sophisticated models for others.

Enables Domain Expert Input

Leadership with domain knowledge can engage with data scientists on feature selection intuitively.

Disadvantages

Prone to Overfitting

With insufficient data or overly complex models, very easy to overfit and memorize noise.

Unstable

Small changes in data can create completely different trees. May arrive at similar decisions via different paths.

Biased with Imbalanced Data

If classes aren't balanced (e.g., loan approvals), the tree can become biased toward majority class.

Often Lower Accuracy

Has poor accuracy compared to other models, especially ensemble methods and neural networks.

May Need Synthetic Data

For imbalanced classes, may need to generate synthetic data for poorly represented class—with its own challenges.

Overfitting Prevention: Pruning

Pre-Pruning

Early Stopping

Stop tree growth before it becomes too complex.

  • Set maximum tree depth
  • Minimum samples required to split
  • Minimum samples per leaf node
  • Maximum number of leaf nodes

Post-Pruning

Grow then Cut

Build full tree first, then remove branches.

  • Cost Complexity Pruning (ccp_alpha)
  • Reduced Error Pruning
  • Remove low-impact branches
  • Cross-validation for validation

Key Concerns

  • Human Bias Transfers: Biased humans labeling data → machines learn that bias
  • Black Box Fears: Concerns about discrimination & opacity in AI decisions
  • Feature Selection Ethics: Moral choices in which features to use

Why Decision Trees Help

Because you can understand the tree structure, you can say "this is unfair" or "I'm not willing to use this feature" and make moral choices. Transparency enables accountability—leadership can engage with data scientists to ensure ethical feature selection.

Key Takeaways

Use Decision Trees When:

  • Explainability is critical
  • Working with small datasets
  • Mixed data types (numeric + categorical)
  • Domain experts need to validate model
  • Regulatory compliance requires transparency

Avoid Decision Trees When:

  • Highest accuracy is required
  • Data is highly imbalanced
  • Model stability is critical
  • Complex non-linear relationships exist
  • Consider ensemble methods instead
