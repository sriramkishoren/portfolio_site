P(A|B)

Posterior Probability

The updated probability of class A given the new evidence B.

P(B|A)

Likelihood

Probability of seeing features B given that the class is A.

P(A)

Prior Probability

Initial belief about class A before seeing any data.

P(B)

Evidence

The total probability of observing the features B. It acts as a normalizing constant.

The "Naive" Assumption

Independence Assumption

Naive Bayes assumes that all features are independent of each other given the class.

P(Feature1, Feature2 | Class)
≈ P(Feature1 | Class) × P(Feature2 | Class)

In reality, this is rarely true (hence "Naive"), but it simplifies computation drastically and often works surprisingly well.

Where it Fails (Dependencies)

  • Time-Series Data (Stock Prices)

    Previous observations influence current ones. Data is not independent over time.

  • Spatial Data (Housing/Zillow)

    Real estate prices depend on neighborhoods. A house price is not independent of its neighbors.

Bayesian Learning Process

01

Prior Assumption

The model starts with an initial guess (prior) about how the data is distributed.

Example: "I think the mean is 5.0"
02

Parameter Updating

"Naive Bayes is designed to update parameters as new data is introduced."

Unlike models that need full retraining, it simply adjusts its estimates as new evidence arrives.

Estimate update: 5.0 → 5.1

Weighting & Boundaries

New data can be weighted differently than old data, causing the decision boundary to shift dynamically.

Dynamic Adaptation

NB Variants

Gaussian NB

For continuous data. Assumes features follow a normal (bell curve) distribution.

Multinomial NB

For counts (e.g., word frequencies). Best for text classification/spam filtering.

Bernoulli NB

For binary data (0/1, Yes/No). Good for checking presence/absence of words.

Decision Boundaries

Limitation

Linear Separation

Naive Bayes is generally limited to linear decision boundaries. It draws straight lines (or hyperplanes) to separate classes.

Simplified 2D view

"If there is more complex separation (curvy spaces), Support Vector Machines (SVM) might perform better."

Implementation Guide 

from sklearn.naive_bayes import GaussianNB
from sklearn.metrics import precision_score, recall_score

# 1. Initialize
model = GaussianNB()

# 2. Train (Very Fast)
model.fit(X_train, y_train)

# 3. Predict
y_pred = model.predict(X_test)

# 4. Evaluate (Crucial!)
print(f"Precision: {precision_score(y_test, y_pred)}")
print(f"Recall: {recall_score(y_test, y_pred)}")

Advantages

  • Simple and fast to implement
  • Computationally efficient (Linear scaling)
  • Great for small datasets
  • Excellent with categorical data

Disadvantages

  • Independence assumption rarely holds
  • Fails on time-series (stock prices)
  • Fails on spatial data (housing)
  • Limited to linear boundaries
