ARCHITECTURE

Dark Factory Architecture

StrongDM's Software Factory — how Level 5 actually works in production with just 3 engineers.

1
Markdown Specifications

3 markdown files describe what the software should do. No traditional code — just specs.

2
Attractor Agent Reads Specs

Open-source coding agent processes specifications and begins autonomous development.

3
AI Writes & Tests Code

Agent develops against digital twin universe (simulated Okta, Jira, Slack, Google services).

4
External Scenarios Evaluate

Behavioral specs stored separately — agent can't see them. Prevents gaming the tests.

5
Production Deployment

Working software ships automatically. 16K lines of Rust, 9.5K lines of Go, 700 lines of TypeScript.

Key Innovation: External Scenarios

Traditional tests live inside the codebase — AI can optimize for passing tests rather than building correct software. StrongDM's scenarios are stored separately as holdout sets, like ML validation data. The agent never sees evaluation criteria and can't game the system.

SKILLS

The Skills Shift

The career ladder is being rebuilt — and most of us haven't noticed yet. The scarcest skill is no longer "can you write clean code quickly" — it's "can you think clearly about what should exist."

What's Changing

Old bottleneck: Implementation speed and developer capacity
New bottleneck: Specification quality and systems understanding

The bottleneck has shifted from "can you write clean code" to "can you think clearly about what should exist and describe it precisely enough that machines can build it."

Skills That Matter Now

  • Systems thinking: Understanding how pieces interact at scale
  • Domain expertise: Deep knowledge of problem space and users
  • Spec articulation: Describing requirements with zero ambiguity
  • Outcome evaluation: Judging whether software solves the real problem
  • Customer intuition: Anticipating edge cases and user needs
  • Architectural judgment: Knowing what will break under load, where security gaps exist
