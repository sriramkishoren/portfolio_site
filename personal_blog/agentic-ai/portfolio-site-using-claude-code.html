Why Claude Code?

Not a chatbot with a code panel — it's a CLI tool that reads your files, runs commands, and commits to git.

Lives in Your Terminal

It reads your files, runs commands, and commits to git. It feels like pairing with a developer who has your entire codebase in their head.

Remembers Your Project

Through a CLAUDE.md file, every conversation starts with Claude already knowing your architecture and conventions.

Works Iteratively

Small, focused requests. Review each change. Accept or reject. That's how real development works.

Vibe Coding vs. Agentic Coding

Most AI coding today is vibe coding. I used an agentic approach with guardrails — Claude plans and implements, but stops at checkpoints for my approval.

TYPICAL

Vibe Coding

1 You describe what you want
2 AI writes code
3 You review & hope it works
Limitation: You're still doing all the thinking about how to break down the work.
MY APPROACH

Agentic Coding

1 AI plans tasks & asks questions
2 AI creates specs & documentation
3 AI implements with checkpoints for approval
4 AI verifies against acceptance criteria
Advantage: I control what gets built; Claude handles how to execute it.

Getting Started

You need Git, a GitHub account, and an Anthropic API key. That's it.

terminal 
# Install Claude Code (macOS via Homebrew)
brew install claude-code

# Create your project
mkdir my-portfolio && cd my-portfolio && git init

# Start Claude Code
claude

How It Plays Out in Practice

Here's how the certifications page feature flowed through the workflow — catching a missing accessibility requirement before any code was written.

You
"I want to add a certifications showcase page."
Claude — Plan Phase
"A few questions: Grid layout, cards, or timeline? How many certifications? Clickable images with lightbox? Dark mode support?"
You
"Grid — 2 hero cards on top, 5 smaller below. Yes lightbox. Yes dark mode."
Claude — Spec Phase
"Spec created. Approve to create TODO?"
You — Catch!
"Add acceptance criteria for Escape key closing the lightbox."
Claude — Updated
"Updated. Approve to create TODO?" ✓
Problems are cheap to fix in a spec, expensive to fix in code. This accessibility requirement was caught before a single line was written.

Best Prompt Patterns

Patterns that consistently produce better results.

Do This

Be specific about locations
"In personal_blog/index.html, update getSortedPosts() to prioritize posts with an 'order' field."
Ask Claude to explore first
"Explore the codebase and understand how blog filtering works before making changes."
Reference existing patterns
"I use Tailwind dark: variants everywhere. Make sure the new page follows the same pattern."
Use templates for common tasks
"In [file], add [feature]. Current behavior: [X]. Desired behavior: [Y]. Stay consistent."

Avoid This

Skipping CLAUDE.md

The difference between Claude knowing your project and guessing.

Mega-prompts

Break features into steps, review each one. Big prompts produce tangled output.

Blind acceptance

Always read the diffs. I caught over-engineered solutions and missing dark mode classes.

Stale documentation

Update CLAUDE.md as the project evolves. Compounding context is the superpower.

Batch work is where AI really earns its keep

"Add SEO metadata to all 27 blog posts — meta tags, Open Graph, JSON-LD structured data." Claude processed all 27 posts in one go. Doing that manually would have been brutal.

Your Safety Net: Git

Commit after every successful change. This is non-negotiable.

git commands 
git diff                    # See what changed
git checkout .              # Undo uncommitted changes
git reset --soft HEAD~1     # Undo last commit, keep changes
git reset --hard abc1234    # Roll back to a specific commit

The Real Takeaway

The skill of working with AI isn't prompting — it's project management.

Claude Code didn't replace my judgment. It amplified my output. Architecture, UX, what to build and what to skip — those decisions were mine. Claude handled the implementation, the boilerplate, and the repetitive work.

Developers who get the most out of Claude Code:

Define boundaries and constraints upfront
Break work into small, reviewable chunks
Maintain living documentation
Review critically instead of accepting blindly
Commit religiously

Start with a CLAUDE.md file. Define your workflow. Iterate in small steps. You'll be surprised how far you get.
