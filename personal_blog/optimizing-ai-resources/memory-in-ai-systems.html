Memory in AI Systems

Types, Architectures, and Strategic Tradeoffs

"Memory is the workspace of AI — understanding the cost of that workspace is key to smart architecture."

Foundation: Memory vs. Compute

Memory

STORAGE & ACCESS

"How much can I hold?"

  • Analogy: The size of your desk. How many papers can you spread out?
  • Metric: Capacity (GB/TB).
  • Holds: Parameters, Model State, Training Data.

Compute

PROCESSING & SPEED

"How fast can I calculate?"

  • Analogy: Speed of your hands working on the papers.
  • Metric: Throughput (FLOPS - Floating Point Ops/Sec).
  • Performs: Math Operations (Multiplication/Addition).

The Interdependence (Bottlenecks)

Compute-Bound:

High memory, low compute speed. Data sits idle waiting to be processed.

Memory-Bound:

High compute speed, low memory speed. Processors sit idle waiting for data to load.

In Training

  • Mem: Holds weights + gradients + optimizer (4× model size) + activations.
  • Comp: Billions of matrix multiplications (forward & backward pass).
  • Typically Compute-Bound (sheer volume of math dominates).

In Inference

  • Mem: Holds weights + KV cache (grows w/ sequence length).
  • Comp: Forward pass only (no back-prop). Much lighter load.
  • Typically Memory-Bound (moving weights to chip is the bottleneck).

Resource Mapping

  • CPU Moderate Compute, Massive Memory (TBs).
  • GPU Massive Parallel Compute (FLOPS), Limited VRAM (16-80GB).
  • TPU Optimized Tensor Compute, High Bandwidth Memory.

Three Core Memory Types

Short-Term Memory (STM)

  • Stores info in hidden states.
  • Analogy: RAM in computers.
  • Use Case: Predicting the next word in a sentence (RNNs).

Working Memory

  • Subset of STM for manipulation & processing.
  • Used for reasoning about inputs.
  • Use Case: Attention mechanisms weighting specific words higher.

  • Extended period storage.
  • Analogy: Hard Drives.

Memory Context: Training vs. Inference

Training Phase

"The Student Cramming"

  • Goal: Learning weights via backpropagation.
  • Memory Load: Massive. Must store Weights + Gradients + Optimizer States + Activations.
  • Key Issue: The "Vanishing Gradient" problem happens here.

Inference Phase

"The Professional Performing"

  • Goal: Applying the model (e.g., iPhone text prediction).
  • Memory Load: Lighter. Stores Weights + Context (KV Cache).
  • Key Issue: Memory Bandwidth & Latency.

Recurrent Neural Networks (RNN)

The sequential processor that introduced "memory" to neural networks.

The Core Concept: "Sequential Memory"

Standard Networks (Feed-Forward)

Like looking at random flashcards. If you see a card with the word "Bank", you don't know if it means a river bank or a money bank because you have no context of what came before.

Recurrent Networks (RNNs)

Like reading a sentence. When you read "He fished by the...", you remember "fished" (context) to understand that "Bank" means river. RNNs pass this context forward as a Hidden State.

How It Works: The "Unrolled" Loop

The "Loop" can be confusing. It's easier to visualize it unrolled over time. The Hidden State ($h$) acts as a courier, carrying the memory from the previous step to the current one.

Mechanism: At Step 2, the network doesn't just see "Cat". It sees "Cat" + the memory of "The". This combination creates the new memory ($h_2$).

The Limitation: Why we moved to LSTM

While the concept is great, in practice, remembering things from 100 steps ago is hard. As the memory gets passed along step-by-step during training, the signal degrades.

Vanishing Gradient The "memory signal" becomes so small (0.0001...) that the network effectively forgets the start of the sentence.
Exploding Gradient The "memory signal" becomes huge, causing the network weights to fluctuate wildly and fail.

Long Short-Term Memory (LSTM)

Solves the vanishing gradient problem using a complex Memory Cell with four distinct gates that control the flow of information.

Forget Gate

Decides what to discard from the cell state (0=forget, 1=retain).

Input Gate

Decides what new information should be stored.

Core

Cell State

The persistent memory backbone regulated by gates.

Output Gate

Determines what to pass to the next time step.

Best For:
Language Modeling Machine Translation Time-Series
The Tradeoff:

More parameters to tune, higher computational cost, and more memory/storage usage than a simple RNN.

Transformers & Self-Attention

The foundation of modern Generative AI. Unlike RNNs, Transformers do not rely on sequential memory.

  • Parallel Processing

    Processes entire sequences simultaneously using GPUs, unlike the step-by-step crawl of RNNs.

  • Self-Attention

    Selectively attends to different features (words/pixels) regardless of distance in the sequence.

  • Speed vs. Cost

    Massive speed advantage, but requires high-end compute power (Cloud/GPUs).

Supplementary Concepts
Architecture

GRU

Gated Recurrent Unit. A simpler LSTM alternative with fewer gates.

LLM

Context Window

The fixed limit of tokens an LLM can "remember" at once.

Optimization

KV Cache

Caching Key-Value pairs in Transformers to speed up inference.

Hybrid

RAG

Retrieval-Augmented Generation. Combining model weights with external DBs.

Hardware

VRAM vs RAM

GPU memory (VRAM) is the critical bottleneck for training large models.

Compression

Quantization

Reducing precision (e.g., INT8) to lower memory footprint.

Efficiency

FlashAttention

IO-aware exact attention that reduces memory complexity.

Agents

Episodic Memory

Storing specific past experiences/conversations for agents.

Executive Takeaways

1.

Memory is a Workspace: More memory enables complexity but costs money. Treat it like a budget item.

2.

LSTMs Fix RNNs: They solve the "vanishing gradient" problem with gated cells, essential for long sequences.

3.

Transformers Changed the Game: By moving to parallel processing (GPUs), they enabled GenAI but raised compute costs.

4.

Context is King: Don't use a bank's architecture for a small prison system. Analyze data volume first.

5.

The Core Question: Stop asking "What is the highest accuracy?" Start asking "What is the performance-to-cost tradeoff?"
